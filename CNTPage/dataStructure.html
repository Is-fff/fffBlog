      # 红黑树的性质（非严格的二叉平衡树）

      1）每个结点或红或黑

      2）根结点是黑色

      3）空叶子结点是黑色

      4）如果一个结点是红色，那么它的子节点是黑色

      5）从任意一个结点出发到叶子结点（NULL结点）经过的黑结点个数相同

      # 哈夫曼树

      哈夫曼树通常以二叉树的形式出现，所以也称最优二叉树，是一类带权路径长度最短的树

      带权路径长度：路径长度*结点权值

      # 构造过程

      已知权值数组[1,4,24,5,6,10,8,31,22,11]

      1.遍历找出两个最小值建树，根节点为权值和

      5

      1 4

      此时1，4权值结点消去，替换为5

      [5,24,5,6,10,8,31,22,11]

      2.拿到新的权值数组，循环步骤1

      # 哈夫曼编码

      哈夫曼编码是一种利用树结构进行数据压缩的方法，其基本思想是以字符的使用频率作为权值构建一棵哈夫曼树，然后利用这棵树对字符进行编码。哈夫曼编码是一种非常有效的数据压缩算法，在数据存储 (opens new window)、传输和处理中有着广泛的应用。

      构造过程：根据结点权值构造哈夫曼树，从哈夫曼树根节点出发，往左子节点走一步得到0，往右子节点走一步得到1，到达目标结点则拼接出哈夫曼编码。

      Object.create(prototype)实现继承

      Object.create(null)生成纯对象，不用担心原型链上的属性污染

      weakMap()，以对象为key的一种Map,可以用来做对象封箱和私有变量封装

      class User{
    this.name = "xjxj";
	this.age = 21
}
const user = new User();
const mySecret = new WeakMap();//使用mySecret来存放私有变量
mySecret.set(user,"123123123");
mySecret.get(user);//获取用户密码，这个变量，就算是打印user也找不到,但是却会随着user一起被回收
//对象封箱，也是一样，用weakMap来藏一个对象，通过另一个对象来获取。
